Guia do arquivo /etc/default/grub

Configuração do GRUB Bootloader nos sistemas Plutão

⚠️ Correção do Caminho

O caminho correto para o arquivo de configuração do GRUB é:

/etc/default/grub

Você digitou "defult" em vez de "default".

O que é o arquivo /etc/default/grub?

Este é o arquivo de configuração principal do GRUB (GRand Unified Bootloader), o gerenciador de inicialização padrão na maioria das distribuições Linux.

Funções principais:

  • Controla opções de inicialização do sistema
  • Define o tempo de espera (timeout) para seleção do sistema operacional
  • Configura a resolução do menu do GRUB
  • Define parâmetros do kernel passados durante o boot
  • Especifica o sistema operacional padrão a ser iniciado

Comandos úteis para trabalhar com o GRUB

Editar o arquivo de configuração:

sudo nano /etc/default/grub
sudo vim /etc/default/grub

Atualizar o GRUB após modificar o arquivo:

# No Ubuntu/Debian e derivados:
sudo update-grub

# No Arch Linux, Fedora, RHEL e derivados:
sudo grub-mkconfig -o /boot/grub/grub.cfg

Visualizar a configuração atual do GRUB:

cat /etc/default/grub

Exemplo de conteúdo do arquivo

Aqui está um exemplo típico do conteúdo do arquivo /etc/default/grub:

# Configuração do GRUB para sistemas Linux

# Define o sistema operacional padrão (0 = primeiro na lista)
GRUB_DEFAULT=0

# Tempo de espera para seleção (em segundos)
GRUB_TIMEOUT=5

# Nome do distribuidor (detectado automaticamente)
GRUB_DISTRIBUTOR=`lsb_release -i -s 2> /dev/null || echo Debian`

# Parâmetros padrão para o kernel Linux
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"

# Parâmetros adicionais para o kernel Linux
GRUB_CMDLINE_LINUX=""

# Habilita suporte a temas gráficos
#GRUB_THEME="/path/to/theme"

# Define a resolução do menu do GRUB
#GRUB_GFXMODE=1920x1080

# Habilita a memória para sistemas com UEFI
#GRUB_GFXPAYLOAD_LINUX=keep

Parâmetros comuns e suas funções

GRUB_DEFAULT

Define qual entrada do menu será selecionada por padrão. Pode ser um número (0 para a primeira entrada) ou o nome da entrada.

GRUB_TIMEOUT

Tempo em segundos que o menu do GRUB ficará visível antes de iniciar o sistema padrão.

GRUB_CMDLINE_LINUX_DEFAULT

Parâmetros adicionais passados ao kernel durante a inicialização. Exemplos comuns:

  • quiet - Reduz a verbosidade durante o boot
  • splash - Exibe tela de inicialização gráfica
  • nomodeset - Desativa configuração automática de vídeo
  • acpi=off - Desativa ACPI (útil para problemas de energia)

GRUB_GFXMODE

Define a resolução do menu gráfico do GRUB. Exemplo: 1920x1080.

⚠️ Aviso importante

Após modificar o arquivo /etc/default/grub, sempre execute o comando de atualização do GRUB para aplicar as alterações. Caso contrário, as mudanças não terão efeito na próxima inicialização.

Exemplo prático: Alterando o tempo de espera

Para aumentar o tempo de seleção do GRUB para 10 segundos:

  1. Abra o arquivo: sudo nano /etc/default/grub
  2. Altere a linha: GRUB_TIMEOUT=5 para GRUB_TIMEOUT=10
  3. Salve o arquivo (Ctrl+X, Y, Enter no nano)
  4. Atualize o GRUB: sudo update-grub
